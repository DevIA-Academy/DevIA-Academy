# DAPS Process

Versión: 1.0.0

---

# Objetivo

Definir la metodología de trabajo utilizada por DevIA Academy para producir contenido educativo de alta calidad.

Este documento establece cómo debe ejecutarse el trabajo, independientemente del tipo de contenido.

---

# Filosofía

Todo contenido debe ser tratado como un producto de software.

Esto significa que:

- existe planificación
- existe desarrollo
- existe validación
- existe control de calidad
- existe documentación
- existe mejora continua

Nunca improvisamos.

---

# Metodología

DAPS se basa en cuatro pilares.

## 1. Calidad

Nunca se publica contenido sin validar.

---

## 2. Repetibilidad

Todo el proceso debe poder repetirse.

Si una persona diferente sigue DAPS deberá obtener un resultado equivalente.

---

## 3. Automatización

Siempre que una tarea pueda automatizarse sin perder calidad deberá automatizarse.

---

## 4. Escalabilidad

El sistema debe funcionar igual para:

- 1 video
- 100 videos
- 1000 videos

---

# Roles

Aunque actualmente el proyecto sea desarrollado por una sola persona, DAPS define responsabilidades.

## Product Owner

Define qué contenido crear.

---

## Technical Lead

Desarrolla y valida el proyecto.

---

## Content Designer

Convierte el desarrollo en material educativo.

---

## QA

Valida:

- precisión técnica
- claridad
- ritmo
- calidad visual

---

## Publisher

Publica y distribuye el contenido.

---

# Artefactos

Cada proyecto genera como mínimo:

- código fuente
- documentación
- guion
- storyboard
- miniatura
- video
- contenido corto

---

# Control de calidad

Todo contenido debe cumplir:

- compila
- funciona
- aporta valor
- explica el por qué
- sigue el Design System
- sigue el Brand Book

---

# Mejora continua

Después de publicar se analizan las métricas.

Los aprendizajes obtenidos deberán incorporarse a futuras versiones de DAPS.
